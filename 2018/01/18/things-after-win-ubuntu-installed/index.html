安装完Win 10 和Ubuntu 17.10之后的那些事儿

好像又好久好久没有更新博客了，如果不是因为要给域名续费也许我都快要忘记这里了吧。不过还好，我回来了。

最近洗心革面，从头做熊，把电脑上的操作系统重新安装了一遍，又顺手装上了最新的Ubuntu。受到Intel 的那个bug的影响，现在的Ubuntu 的最新版本号已经是17.10.1了，刚刚重新上架，新鲜出炉。

这篇文章主要是记录一下装完双系统，尤其是Ubuntu 之后遇到的一些问题和解决办法以及一些Ubuntu 的简单设置。

1. Ubuntu 下 Bluetooth 和WiFi 冲突导致网速极慢的解决办法

安装完Ubuntu 之后遇到的第一个大问题可能就是这个网速慢到令人发指了，没连蓝牙鼠标之前还好，但是连完之后就仿佛被限速了一般。刚开始的时候没有反应过来是鼠标的问题。尝试了各种方法，甚至又重装了一遍之后才意识到是蓝牙导致的无线网速过慢了。

解决办法：

  1. 打开并修改/etc/modprobe.d/iwlwifi.conf 文件

    sudo nano /etc/modprobe.d/iwlwifi.conf

  2. 在文件尾加入下面这行代码

    options iwlwifi bt_coex_active=0 swcrypto=1 11n_disable=8

  3. 保存，重启，问题解决

2. 解决Ubuntu 与 Windows 时间不同步（Windows 时间错乱的问题）

这个大概也算是一个常见的问题了，涉及到两个系统采用了不同的UTC 和GMT 时间标准的问题。在Ubuntu 更新到17之后以前的修正方式似乎不能用了，两种方法都写到这里吧。

原方法：

  1. 打开文件/etc/default/rcs

    sudo nano /etc/default/rcs

  2. 找到 utc = yes 这一行并修改为

    utc = no

现方法：

  1. 输入命令：

    timedatectl set-local-rtc true

  2. 重启

3. 罗技 MX Master 鼠标在Ubuntu 下的自定义按键问题

MX Master 可以说是广受好评的一款鼠标了吧，在Windows 下可以说是非常好用的一款了，上面大大小小带了差不多七个按键和两个滚轮，在Linux 下不充分利用起来，可惜了。下面就说说这些按键的自定义方法吧。

  1. 打开终端并安装

    sudo apt-get install xbindkeys xautomation x11-utils

  2. 运行命令

    xev

  3. 这时会出现一个白色的窗口，把光标移动到上面会在终端中出现鼠标的信息，按下想要自定义的按键，并且记录下这个按键的代号。例如按下鼠标左键后，会出现

    ButtonPress event, serial 34, synthetic NO, window 0x3800001, root 0x273, subw 0x0, time 3836829, (68,88), root:(185,195), state 0x0, button 1, same_screen YES

    注意button 后面的数字就是按键的代号，这里鼠标左键对应 ‘1’

  4. 创建配置文件并打开

    xbindkeys --defaults > $HOME/.xbindkeysrc

    sudo nano $HOME/.xbindkeysrc

  5. 根据需求修改配置文件。这里我以我的使用习惯举例，我定义了MX Master的两个侧按键，这两个按键对应的代号是8 和9，我将一个设置为关闭当前标签（Ctrl + W），将另一个设置为粘贴（Ctrl + V）。具体代码如下，其他的可以举一反三了。

    "xte 'keydown Control_L' 'key w' 'keyup Control_L'"

    b:9

    "xte 'keydown Control_L' 'key v' 'keyup Control_L'"

    b:8

4. Oracle JDK 的安装和环境变量的设置

既然打算把以后的开发工作都放到Linux 上，那首先Oracle JDK的安装是必不可少的了，简单说一下安装和环境变量的设置吧。

4.1 安装Oracle JDK

  1. 添加Oracle JDK的PPA：

    sudo add-apt-repository ppa:webupd8team/java

    sudo apt-get update

  2. 安装JDK 9

    sudo apt-get install oracle-java9-installer

  3. 选择将起设置为默认的JDK

    sudo update-alternatives --config java

4.2 配置JDK环境变量

  1. 查询并复制路径（形如’/usr/lib/jvm/java-9-oracle/bin/java‘）

    sudo update-alternatives --config java

  2. 打开环境变量配置文件

    sudo nano /etc/environment

    并在文件尾添加刚才复制的路径

    JAVA_HOME="/usr/lib/jvm/java-9-oracle"

  3. 保存退出并重载该配置文件

    source /etc/environment

  4. 可以通过下面的这个命令来查看环境变量是否设置成功

    echo $JAVA_HOME
