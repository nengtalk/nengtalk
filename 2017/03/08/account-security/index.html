我们要如何面对那些已经不再安全的密码

凸凹熊 3月 08, 2017

前两天手贱，点了一个不该点的软件（想尝试安装一个国外的破解版IDM，大意了……）然后电脑就中毒了……埋得很深，副作用很强的一个木马，Chrome的数据被清空了，电脑CPU使用率超高，一直在后台自动安装垃圾软件……没办法，使用了两个国内外的杀毒软件，发现连流氓都无法收拾流氓之后，之后还原了电脑。联想到最近看到的两条新闻，今天就说说账号安全那些事吧。

那些我们经常会听到看到的新闻

最近有两条新闻，可能我把这篇坑填完发出来的时候应该都算是旧闻了，一个是京东新招聘的安全部门的试用期员工大手一挥卖掉了50亿条公民信息，另一个是网传迅雷的用户数据库被拖库了，有可能导致大量的用户信息泄露。其实这些消息已经不是个案了，而且当这些消息传出来的时候，大部分的数据库内容都早已经被黑客们吸收消化的差不多了。都说现在的互联网已经没有什么隐私可言，不过，我们也还是不能破罐子破摔吧，反正做不到早发现，尽力做到早预防吧。

要怎样去管理我们的密码

那么，要怎样去管理我们的密码呢？

拿着生日当密码
拿着姓名+生日当密码
拿着手机号当密码
这样做密码被猜出来和撞库的概率还是蛮高的

一套密码用来给小网站社区
一套密码用来给大网站社交应用
一套密码给网银和支付工具
这样做看着还行，但是，当大网站的数据被泄露了呢？比如当年163邮箱的数据泄露……

一套纯数字的密码
一套数字+字母的密码
一套数字+大小写字母区分的密码
这样的做法其实大部分是为了满足一些网站的要求组出来的吧……

我觉得上面说的几种方案都属于很常见的吧，虽然方便，但不得不说还是很不安全的。接下来，就说几种更加安全的密码和密码管理方案吧

使用密码管理工具

这类工具其实还是蛮多的，像Lastpass，1Password，Keepass等等，现在这些工具基本都推出了多平台的对应版本，这类工具的好处在于他们可以针对不同网站随机生成强密码，你只需要记住一个密码就可以了。好用而且安全，当然，千万不要泄露主密码。

我常用的工具是Lastpass，虽然不是很想推他，因为我花了很多钱买了一年会员准备在移动端使用的时候，他将移动端免费了……

纠结想用哪个的，不妨都下载下来试试

开启两步验证

两部验证看起来虽然麻烦，但很大程度上增强了安全性。国内外的两步验证采用了不同的形式，不过算是殊途同归吧。国内大多数网站用的是手机号发送短信验证码的形式，比如163邮箱；而国外大多数采用的是在绑定的手机端随机生成验证码并且定时变化，有些像国内的银行电子口令卡，比如苹果，谷歌，微软等等。我以前也曾觉得每次登陆个账号还要去手机上看验证码，觉得很麻烦，但慢慢的也就已经习惯了。另外推荐一个叫做Authy 的软件，这个可以同步管理大多数支持两部验证的网站，并且支持云同步，也就不怕更换设备了。当然你需要给Authy 设置一个在别处没有使用过的强密码。

预备多套密码

如果你不喜欢用密码管理工具，也不喜欢两步验证，那么就只好预备多套密码，并且定时更换了。记得避免使用常见的姓名手机号生日之类的组合，这些很容易会被猜出来的。另外，你可以增加一些特定的前缀来增加密码的复杂性，比如给淘宝的账号前缀加个jd，给糯米的密码前缀加个mt（美团），总之，要够独特，而且，重要的密码要定时更换，尤其是在更换手机号的时候。

不要把账号借给别人

不要，千万不要把账号借给别人，尤其是icloud的账号，Google的账号以及你同步了个人数据的一些账号，也许别人只是想借你的账号下载一个应用程序，一个游戏，但没准不经意的操作就顺手把你的密码，联系人，备忘录什么的都同步过去了，更一不留神，没准账户的一些重要财产也跟着就过去了。注册一个账号又不费什么功夫，如果不得不借的话，记得先换一个临时密码。

谨慎处理验证码

骗子的花样越来越高级，骗取验证码的技巧也越来越高级。总之，对待验证码的态度要非常的坚决，不给人，不乱输。看好每条验证码的发送方是谁，朋友跟你要验证码的话，一定要至少语音确认朋友的身份。对于自己的财产相关的账户，能开启短信通知变动通知的不要吝惜每个月几块钱的短信服务费。

以上就是我的认知范围内的一些关于账号安全的见解，如果你有什么更好的方法，不妨一起分享出来。

如果有一天，我们突然不在……

其实我思考这个问题已经很久了，如果有一天，我们突然不在，那么我们的这些账号要怎么处理呢？也许我们在一些网站里写过很多没发过的重要的内容，也许在一些理财平台还有过一些投资，也许我们在一些保险平台还买过一些保险可以去理赔……

可是我们又不敢，或者还没来得及把这些东西告诉我们的亲人，那如果某一天突然的来临又会怎么办呢？

也许亲人可以通过申诉，费劲周折取得你的部分网站的管理权限，但他们也许也不知道你到底还有多少其他地方的账号密码。

也许这也是我推荐大家使用密码管理工具的另一个重要的原因，不妨把你的最重要的那个密码留给你最亲近的人，或者通过某种途径，能让你最亲近的人获得那个密码……
